# GitHub代理工具 - 使用指南

## 工具简介

这是一个专业的GitHub代理工具，帮助您轻松访问GitHub网站，特别是GitHub设备登录和确认页面。

## 文件说明

1. **GitHub代理工具最终版.py** - 核心Python脚本，包含所有功能实现
2. **启动GitHub代理工具.bat** - 快速启动批处理文件
3. **github_simple_proxy.py** - 简易版代理工具（备用）
4. **GitHub简易代理工具.bat** - 简易版启动脚本（备用）

## 使用方法

### 方法一：通过批处理文件启动（推荐）

1. **以管理员身份运行**：
   - 找到 `启动GitHub代理工具.bat`
   - 右键点击，选择 **以管理员身份运行**
   - 这将确保所有功能（特别是代理设置）正常工作

2. **普通运行**：
   - 直接双击 `启动GitHub代理工具.bat`
   - 注意：部分功能（如系统代理设置）需要管理员权限

### 方法二：直接运行Python脚本

1. 确保您已安装Python 3.6或更高版本
2. 右键点击 `GitHub代理工具最终版.py`
3. 选择 **以管理员身份运行** 或直接双击

## 常见问题

### Q: 为什么需要管理员权限？
A: 修改系统代理设置需要管理员权限才能确保在所有浏览器中生效。

### Q: 代理设置后仍无法访问GitHub怎么办？
A: 请尝试：
1. 清除浏览器缓存和Cookie
2. 重启浏览器
3. 尝试使用其他浏览器
4. 重新测试并选择其他代理

### Q: 所有代理都不可用怎么办？
A: 代理服务器可能已更新或被屏蔽，请：
1. 检查您的网络连接
2. 尝试手动添加其他代理
3. 考虑使用商业VPN服务

## 注意事项

- 代理服务器可能会定期变化，请定期运行测试
- 使用代理时请注意网络安全
- 建议在不需要访问GitHub时禁用代理
- 代理服务的可用性取决于网络环境和服务器状态
